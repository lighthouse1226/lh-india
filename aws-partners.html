Refactoring for Move

In planning a migration, it may be uncertain whether some applications can be moved. Even relatively modern applications were not built for a public cloud deployment and may lack the security features necessary. An investigation of the code base and architecture can determine if it is possible and what is required to make the application movable.

Refactoring for Migration

The full benefit of migrating to AWS is to re-engineer the application to be cloud native, taking advantage of features like auto-scaling and self-healing. Refactoring an application to incorporate these capabilities may be possible after full investigation of the code base and architecture.

Replatforming for Migration

Legacy applications built on older platforms and tools most often need to be rewritten from the ground up. There is a real opportunity to build the new application to be fully cloud native, incorporating current technologies for autoscaling, redundancy and include the highest level of security and flexibility. By skillfully reverse engineering the legacy application, the original capabilities can be retained in the new application and allow for deployment within AWS.
